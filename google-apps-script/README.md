# Google スプレッドシート連携 セットアップ手順

このアプリは Supabase の代わりに **Google スプレッドシート** をバックエンド（データベース＋認証）として使えます。
無料・追加費用なしで、社員アカウント／権限／日報／サンクスメッセージ／事例をすべてシートに保存します。

---

## 1. スプレッドシートを用意する

1. [Googleスプレッドシート](https://sheets.new) を新規作成（名前は「ライフアップアプリDB」など任意）
2. メニューの **拡張機能 → Apps Script** を開く
3. デフォルトのコードを全部消し、`Code.gs` の中身をすべて貼り付けて保存（💾）

> `users` `reports` `thanks` `cases` の各タブは初回アクセス時に自動で作成されます。

---

## 2. シークレットキーを設定する

1. Apps Scriptエディタ左の **⚙️ プロジェクトの設定** を開く
2. 下部「スクリプト プロパティ」→ **スクリプト プロパティを追加**
   - プロパティ: `API_SECRET`
   - 値: 任意の長いランダム文字列（例: パスワード生成ツールで作った40文字程度）
3. 保存

---

## 3. ウェブアプリとしてデプロイする

1. 右上 **デプロイ → 新しいデプロイ**
2. 歯車⚙️ → **ウェブアプリ** を選択
3. 設定:
   - 説明: 任意
   - **次のユーザーとして実行: 自分**
   - **アクセスできるユーザー: 全員**
4. **デプロイ** をクリック → 初回は権限承認を求められるので許可
5. 表示される **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec`）をコピー

---

## 4. アプリ（Vercel）に環境変数を設定する

Vercel のプロジェクト設定 → Environment Variables に以下を登録：

| 変数名 | 値 |
|--------|-----|
| `SHEETS_API_URL` | 手順3でコピーしたウェブアプリURL |
| `SHEETS_API_SECRET` | 手順2で設定した `API_SECRET` と**同じ文字列** |
| `SESSION_SECRET` | 別の長いランダム文字列（ログインCookieの署名用） |

設定後、再デプロイすれば連携完了です。

---

## 動作の流れ

- **最初にアカウント登録した人 → 自動で「管理者(manager)」** になります
- 以降の登録者は「一般(general)」
- 管理者はサイドバーの **「ユーザー管理」** から各メンバーの権限（管理者／リーダー／一般）を変更できます
- 日報・サンクス・事例の投稿はすべてスプレッドシートの各タブに記録され、経験値も自動加算されます

> 環境変数が未設定の場合は **デモモード**（ログインは誰でも通り、データは保存されない）で動作します。

---

## セキュリティについて

- パスワードは **scrypt でハッシュ化**してから保存（平文では保存されません）
- `API_SECRET` を知らないとスプレッドシートAPIは一切応答しません
- スプレッドシート自体は共有しない限り作成者のみ閲覧可能です
- ログインセッションは **署名付きCookie（HMAC-SHA256）** で管理されます
